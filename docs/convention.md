# Convention

## 코딩 컨벤션 (중요)

- 꼭 필요한 최소한의 코드만 작성한다.
- 중복 없이, 간결하고 명확하게 쓴다.
- 변수, 함수 이름은 역할이 직관적으로 드러나게 짓는다.
- 역할 단위로 코드를 분리하고, 책임은 명확히 나눈다.
- 타입스크립트의 타입 추론을 우선하고, 불필요한 명시는 지양한다.
- 읽기 쉬운 코드를 우선하고, 성능 최적화는 효과가 확실할 때만 적용한다.
- 함수/선언형 패턴을 선호하되, 유지보수성과 균형을 고려한다.
- 컴포넌트와 파일은 단순하고 재사용 가능하게 만든다
- 로깅과 에러 처리는 공통화하고, 디버깅이 쉬워야 한다.
- 스케줄, 타이머, 분기 로직은 흐름이 명확하게 보이도록 작성한다.
- 데이터 구조는 중복 없이 확장 가능하게 설계한다.
- 리팩토링 시 중복은 함수로 분리하고, 의도가 드러나게 정리한다. 불필요한 추상화는 피한다.

### 추상화 원칙

- 기존 구조를 먼저 확인하고, 패턴을 따라간다.
- 같은 역할의 코드가 있으면 재사용 또는 확장한다.
- 중복 제거 또는 복잡도 완화의 실익이 있을 때만 추상화한다.
- 이름, 위치, 책임 분리는 기존과 일관되게 맞춘다.
- 추상화는 단순하고 명확하게 작성한다.
- 의미 없는 래핑이나 1회용 추상화는 금지한다.
