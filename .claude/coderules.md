# 바이브 코딩 컨벤션

## 핵심 4원칙 (우선순위 순) CRITICAL

1. 기존 구조 먼저 확인 - 패턴 따르기
2. 필요한 것만 구현 - 20년차처럼 최소한으로
3. 추상화는 신중하게 - 확실할 때만, 단순하게
4. 안 쓰는 코드 삭제 - 깔끔하게 정리

## 코드 품질

- 최소한의 코드, 중복 없이 간결하게
- 변수/함수명은 역할이 직관적으로 드러나게
- 역할 단위로 분리, 책임 명확히 구분
- 가독성 우선, 성능 최적화는 근거 있을 때만
- 함수형·선언형 패턴 선호하되 유지보수성과 균형

## 아키텍처

- 데이터 구조는 중복 없이 확장 가능하게, SSOT 원칙 준수
- 로깅과 에러 처리는 공통화해 디버깅 쉽게
- 스케줄·타이머·분기 로직은 흐름이 명확하게 보이도록
- 컴포넌트/파일은 단순하고 재사용 가능하게

## 언어별

- Typescript: 타입 추론 우선, 불필요한 명시 지양

## 체크포인트

- 기존 패턴 따름?
- 불필요한 코드 없음?
- 최소한으로 구현?
- 추상화 필요성 명확?
- 변수/함수명 직관적?
- 책임 분리 명확?
